# EP Cloud / Enterprise / Vertical Packs: Commercial Brief

## Open Protocol Layer

Emilia Protocol is open source, forkable, and has no vendor lock-in.

- MIT-licensed protocol specification and reference implementation
- Organizations can self-host, fork, or build on top without licensing fees
- No proprietary wire formats, token structures, or policy languages
- Interoperable by design: any conforming implementation works with any other

The protocol is the foundation. The commercial products build operational
infrastructure on top of it.

## EP Cloud

Managed infrastructure for organizations that adopt the open protocol.

| Capability | Description |
|------------|-------------|
| Policy Registry | Central management of action policies, risk classes, and signoff thresholds |
| Signoff Orchestration | Routes signoff requests to named principals via configured channels |
| Event Explorer | Search, filter, and inspect every action decision in real time |
| Audit Exports | Structured export of evidence records for compliance review |
| Tenant Controls | Multi-tenant isolation, per-tenant policy, per-tenant retention |
| Webhooks | Real-time notifications on policy violations, signoff events, blocked actions |

**Revenue model:** Recurring infrastructure subscription. Priced per tenant
and per action volume.

## EP Enterprise

For organizations requiring private deployment and regulatory controls.

| Capability | Description |
|------------|-------------|
| VPC Deployment | Runs entirely within the customer's cloud account |
| Data Residency | Evidence and policy data remain in specified geographic regions |
| SSO / SCIM | Integration with enterprise identity providers |
| Evidence Retention | Configurable retention periods with legal hold capability |
| Dedicated Support | Named support contacts, SLA-backed response times |

**Revenue model:** Annual enterprise license plus deployment support.

## Vertical Packs

Pre-configured policy sets, signoff workflows, and evidence templates for
specific regulatory environments.

### Government Pack
- Benefits fraud prevention policies (payment destination change, eligibility override)
- IG/GAO audit evidence templates
- FedRAMP-aligned deployment configuration
- Operator override detection and signoff enforcement

### Financial Pack
- Wire transfer and beneficiary change policies
- SOX evidence generation and retention
- Dual signoff for treasury operations
- BEC attack pattern detection rules

### Agent Governance Pack
- Risk class definitions for autonomous agent actions
- Signoff thresholds by action severity
- EU AI Act accountability chain evidence
- Human-in-the-loop enforcement at configurable decision points

## Competitive Position

| Layer | Function |
|-------|----------|
| Open protocol (bottom) | Free adoption, no switching cost, community contribution |
| Managed control plane (top) | Operational infrastructure that organizations pay for |

The open protocol drives adoption. The managed layer captures recurring revenue.
Compliance requirements and liability reduction are non-discretionary budget items
for government, financial, and regulated enterprise buyers. Organizations that
adopt the protocol for one workflow expand to additional action classes over time.
