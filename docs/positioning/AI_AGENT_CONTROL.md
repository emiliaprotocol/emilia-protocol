# EP for AI / Agent Control

As AI systems move from recommendation to execution, the missing governance layer is action control.

EMILIA Protocol (EP) is the trust-control layer between intent and execution. It does not judge whether a model is “smart.” It enforces whether a high-risk action is authorized, exactly scoped, policy-bound, replay-resistant, and auditable before it proceeds.

## Where this matters

- account changes
- procurement actions
- system configuration changes
- delegated approvals
- customer-impacting service actions
- autonomous or semi-autonomous tooling
- sensitive integrations and tool use

## Why EP fits

EP gives agent workflows:
- actor identity anchoring
- authority chain checks
- exact action binding
- policy version and hash binding
- one-time consumption
- immutable event traceability

## Positioning line

MCP tells agents how to use tools. EP tells systems whether a high-risk action should be allowed to proceed.
