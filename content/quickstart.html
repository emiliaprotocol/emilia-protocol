Quickstart

Get EP integrated in 5 minutes.

Add the MCP server to your Claude config, call the REST API directly, or use the TypeScript/Python SDK. Everything is open — no approval required.

Base URL: https://emiliaprotocol.ai  ·  Auth: Most write endpoints require Authorization: Bearer ep_live_.... Entity registration is public and returns your first API key  ·  Read endpoints are public — no API key needed.
Local development: Use http://localhost:3000 as your base URL. The Docker stack runs the app container — you must provide NEXT_PUBLIC_SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY pointing at a real Supabase project.

Step 1 — Add the MCP Server

1
Plug EP into any MCP client
~60 seconds

Add this to your claude_desktop_config.json (or any MCP-compatible client config). EP gives your agent 29 trust tools instantly.

JSON — claude_desktop_config.json
{
  "mcpServers": {
    "emilia-protocol": {
      "command": "npx",
      "args": ["-y", "@emilia-protocol/mcp-server"],
      "env": {
        "EP_API_KEY": "ep_live_..."  // returned when you register your entity
      }
    }
  }
}

Without EP_API_KEY, read-only tools still work: ep_trust_profile, ep_trust_evaluate, ep_install_preflight, ep_search_entities, ep_leaderboard.

29 MCP tools available: ep_trust_profile, ep_trust_evaluate, ep_submit_receipt, ep_search_entities, ep_verify_receipt, ep_register_entity, ep_leaderboard, ep_dispute_file, ep_dispute_status, ep_report_trust_issue, ep_appeal_dispute, ep_install_preflight, ep_principal_lookup, ep_lineage, ep_list_policies, ep_create_delegation, ep_verify_delegation, ep_trust_gate, ep_batch_submit, ep_configure_auto_receipt, ep_domain_score, ep_generate_zk_proof, ep_verify_zk_proof, ep_delegation_judgment, ep_issue_commit, ep_verify_commit, ep_get_commit_status, ep_revoke_commit, ep_bind_receipt_to_commit

Step 2 — Call the REST API

2
Direct HTTP — no SDK needed
~30 seconds
Shell
# Install preflight for an MCP server (public, no auth)
$ curl -X POST https://emiliaprotocol.ai/api/trust/install-preflight \
  -H "Content-Type: application/json" \
  -d '{"entity_id":"mcp-server-ep-v1","policy":"mcp_server_safe_v1"}'

# Get a trust profile
$ curl https://emiliaprotocol.ai/api/trust/profile/rex-booking-v1

# Evaluate against a policy
$ curl -X POST https://emiliaprotocol.ai/api/trust/evaluate \
  -H "Content-Type: application/json" \
  -d '{"entity_id":"rex-booking-v1","policy":"standard"}'

# Search entities
$ curl "https://emiliaprotocol.ai/api/entities/search?q=booking&type=merchant"
TypeScript
import { EPClient } from '@emilia-protocol/sdk';

const ep = new EPClient({
  apiKey: process.env.EP_API_KEY,               // optional for reads
  baseUrl: 'https://emiliaprotocol.ai',
});

// Install preflight — check before installing any MCP server
const preflight = await ep.installPreflight('mcp-server-ep-v1');
console.log(preflight.decision); // 'allow' | 'review' | 'deny'

// Get a trust profile
const profile = await ep.getTrustProfile('rex-booking-v1');
console.log(profile.current_confidence); // 'confident' | 'emerging' | 'provisional' | 'insufficient' | 'pending'

// Evaluate against a policy
const result = await ep.evaluateTrust({
  entityId: 'rex-booking-v1',
  policy: 'standard',
  context: { category: 'electronics', value_band: 'high' },
});
if (!result.pass) console.log('Failures:', result.failures);
Python
from emilia_protocol import EPClient

ep = EPClient(
    api_key="ep_live_...",           # optional for reads
    base_url="https://emiliaprotocol.ai"
)

# Install preflight — check before installing any MCP server
preflight = ep.install_preflight("mcp-server-ep-v1")
print(preflight["decision"])  # allow | review | deny

# Get a trust profile
profile = ep.get_trust_profile("rex-booking-v1")
print(profile["current_confidence"])  # confident | emerging | provisional | insufficient | pending

# Evaluate against a policy
result = ep.evaluate_trust("rex-booking-v1", policy="standard",
                     context={"category": "electronics"})
if result["decision"] != "allow":
    print("Reasons:", result["reasons"])

Step 3 — Register Your Entity

3
Get your entity ID and API key
~1 minute

Register your agent, merchant, or service. You'll get a permanent entity_id and an API key for write operations. Save the API key — it's only shown once.

Shell
$ curl -X POST https://emiliaprotocol.ai/api/entities/register \

  -H "Content-Type: application/json" \
  -d '{
    "entity_id": "acme-merchant-v1",
    "display_name": "ACME Merchant",
    "entity_type": "merchant",
    "description": "Electronics marketplace seller",
    "capabilities": ["retail", "electronics", "returns"]
  }'

# Response
{
  "entity": { "entity_id": "acme-merchant-v1", ... },
  "api_key": "ep_live_..."   // Save this — shown once
}
Entity types: Core types include agent, merchant, and service_provider. EP also supports software counterparties: github_app, github_action, mcp_server, npm_package, chrome_extension, shopify_app, marketplace_plugin, and agent_tool. The entity_id is permanent and lowercase-hyphenated.

Step 4 — Submit a Receipt

4
Record every transaction outcome
~2 minutes

After every interaction with a counterparty, submit a receipt. The agent_behavior field is the strongest signal — it records what you did next, not what you thought of the experience.

JavaScript
const response = await fetch('https://emiliaprotocol.ai/api/receipts/submit', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${EP_API_KEY}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    entity_id: 'acme-merchant-v1',     // who you transacted with
    transaction_ref: 'order_8f7a2b9c',   // your external reference (required)
    transaction_type: 'purchase',        // purchase|service|task_completion|delivery|return
    agent_behavior: 'completed',        // STRONGEST SIGNAL: completed|retried_same|retried_different|abandoned|disputed
    delivery_accuracy: 95,              // 0-100 (optional)
    product_accuracy: 92,               // 0-100 (optional)
    price_integrity: 100,               // 0-100 (optional)
    context: {                            // context key for context-aware evaluation
      category: 'electronics',
      geo: 'us',
      value_band: 'high',               // micro|low|medium|high|very_high
    },
  }),
});
const { receipt } = await response.json();
console.log(receipt.receipt_id);   // ep_rcpt_...
console.log(receipt.receipt_hash);  // SHA-256 chain-linked hash
Why agent_behavior is the most important field: It records what your agent did after the transaction — came back (completed), switched to a competitor (retried_different), or abandoned the flow. This behavioral signal carries 40% of the trust weight because it's hard to fake and directly measures value delivery.

Step 5 — Query a Trust Profile

5
Read behavioral trust before transacting
~1 API call
JavaScript
const r = await fetch('https://emiliaprotocol.ai/api/trust/profile/acme-merchant-v1');
const profile = await r.json();

// Key fields
profile.current_confidence   // 'pending' | 'insufficient' | 'provisional' | 'emerging' | 'confident'
profile.effective_evidence_current  // credibility-weighted evidence score
profile.trust_profile.behavioral.completion_rate  // 0-100
profile.trust_profile.behavioral.dispute_rate     // 0-100
profile.disputes.total         // total disputes against this entity
profile.anomaly               // present if trust velocity anomaly detected

Step 6 — Evaluate Against a Policy

6
Get a Trust Decision (allow/review/deny) with reasons
~1 API call

Instead of reading raw numbers and writing your own logic, pass a policy name and get back a Trust Decision (allow, review, or deny) with specific reasons.

JavaScript
const r = await fetch('https://emiliaprotocol.ai/api/trust/evaluate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    entity_id: 'acme-merchant-v1',
    policy: 'standard',   // strict | standard | permissive | discovery
    context: { category: 'electronics', value_band: 'high' },
  }),
});
const result = await r.json();

if (result.pass) {
  // proceed with transaction
} else {
  console.log(result.failures);  // ["Dispute rate too high (12% > 5%)", ...]
  console.log(result.warnings);  // non-blocking observations
}
strict
High-value transactions. Requires confident confidence + very low dispute rate.
standard
Normal transactions. Requires emerging confidence + reasonable behavioral rates.
permissive
Low-risk interactions. Allows provisional confidence. Excludes only clear bad actors.
discovery
First-time interactions. Allows unevaluated entities. Use for new counterparties.

AI Agents — Integration Pattern

The canonical pattern for agents: check trust profile before routing, submit receipt after every transaction, use install preflight before installing any software.

TypeScript — Agent trust pattern
// Before transacting with any counterparty
async function shouldTransactWith(entityId: string, context: Context) {
  const evaluation = await ep.evaluate(entityId, {
    policy: context.value_band === 'high' ? 'strict' : 'standard',
    context,
  });
  return { allowed: evaluation.pass, reasons: evaluation.failures };
}

// After completing a transaction
async function recordTransaction(entityId: string, outcome: Outcome) {
  await ep.submitReceipt({
    entity_id: entityId,
    transaction_ref: outcome.ref,
    transaction_type: 'purchase',
    agent_behavior: outcome.cameBack ? 'completed' : 'retried_different',
    context: outcome.context,
  });
}

// Before installing any plugin or MCP server
async function canInstall(entityId: string) {
  const check = await ep.installPreflight(entityId, { policy: 'mcp_server_safe_v1' });
  if (check.decision === 'deny') throw new Error(`Install blocked: ${check.reasons.join(', ')}`);
  return check.decision === 'allow';
}

Platforms — Webhook Integration

For Shopify and other commerce platforms, configure webhooks to auto-submit receipts on every order lifecycle event.

JavaScript — Shopify webhook handler
// POST /webhooks/shopify/orders — handles Shopify order lifecycle
export async function POST(req) {
  const order = await req.json();
  const merchantId = `shopify-${order.vendor.toLowerCase().replace(/\s+/g, '-')}-v1`;

  const behavior =
    order.financial_status === 'refunded' ? 'disputed' :
    order.fulfillment_status === 'fulfilled' ? 'completed' : 'retried_same';

  await fetch('https://emiliaprotocol.ai/api/receipts/submit', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${EP_API_KEY}`, 'Content-Type': 'application/json' },
    body: JSON.stringify({
      entity_id: merchantId,
      transaction_ref: order.id,
      transaction_type: 'purchase',
      agent_behavior: behavior,
      context: { category: order.product_type, geo: order.shipping_address?.country_code?.toLowerCase() },
    }),
  });
  return new Response('ok');
}

Software & MCP Servers — Install Preflight

Use install preflight before adding any MCP server, GitHub App, npm package, or browser extension. EP evaluates publisher identity, permission class, and behavioral trust history.

Shell — Software preflight checks
# Before installing an MCP server
$ curl -X POST https://emiliaprotocol.ai/api/trust/install-preflight \
  -d '{"entity_id":"mcp-server-ep-v1","policy":"mcp_server_safe_v1"}'

# Before installing a GitHub App
$ curl -X POST https://emiliaprotocol.ai/api/trust/install-preflight \
  -d '{"entity_id":"github_app:acme/code-helper","policy":"github_private_repo_safe_v1"}'

API Reference

All endpoints are at https://emiliaprotocol.ai. GET endpoints are public. POST write endpoints require Authorization: Bearer ep_live_.... Exception: /api/entities/register is public and returns the API key.

Full trust profile with behavioral rates, signals, provenance, disputes, and anomaly state
POST
/api/trust/evaluate
Evaluate entity against a policy. Returns a Trust Decision (allow/review/deny) with specific failure reasons
POST
/api/trust/install-preflight
Software install safety check. Returns allow/review/deny with reasons

Trust Policies

Pass a policy name to ep_trust_evaluate or /api/trust/evaluate to get a Trust Decision (allow/review/deny). Four built-in policies plus four software-specific policies.

strict
High-value transactions. Requires confident confidence level.
standard
Normal use. Requires emerging or better.
permissive
Low-risk. Allows provisional confidence. Excludes clear bad actors.
discovery
First-time interactions. Allows unevaluated entities.
mcp_server_safe_v1
MCP servers. Publisher verified + no dangerous permissions.
github_private_repo_safe_v1
GitHub Apps with private repo access. Strict publisher checks.
npm_buildtime_safe_v1
npm packages at build time. Trusted publishing + provenance required.
browser_extension_safe_v1
Chrome extensions. Permission class + behavioral trust history.
