Interactive Demo

Trust Evaluation in 5 Steps

See EMILIA Protocol against live API surfaces. This demo hits the current reference endpoints for trust profiles, policy evaluation, receipts, disputes, and verification.

1
Query a Trust Profile
The canonical output of EP is a trust profile — not a number. It includes behavioral rates, signal breakdowns, provenance composition, confidence level, and dispute history.
GET /api/trust/profile/:entityId
// Try a registered entity GET /api/trust/profile/mcp-server-ep-v1
Calls the real EP API
Why this matters: Agents and systems don't check "score > 70." They evaluate trust profiles against policies. A trust profile tells you why an entity is trusted, not just how much.
2
Evaluate Against a Policy
EP doesn't use arbitrary thresholds. Agents evaluate counterparties against structured policies: strict for high-value, standard for normal, permissive for low-risk, discovery for exploring.
POST /api/trust/evaluate
// Evaluate against "strict" policy { "entity_id": "mcp-server-ep-v1", "policy": "strict" }
Evaluates against the strict trust policy
Why this matters: Different contexts need different trust thresholds. A plugin accessing read-only data needs less trust than one accessing private repos. Policies make this explicit.
3
Run Install Preflight
MCP tells you what a server can do. EP tells you whether it should be trusted enough to connect. Install preflight evaluates software entities against context-specific safety policies.
POST /api/trust/install-preflight
// Should I install this MCP server in a private workspace? { "entity_id": "mcp-server-ep-v1", "policy": "mcp_server_safe_v1", "context": { "host": "mcp", "data_sensitivity": "private_workspace", "tool_scope": "repo_read" } }
Real install safety evaluation
Why this matters: The future trust question for software isn't "is this popular?" — it's "is this safe enough for this policy and scope?"
4
File a Trust Report
A real trust layer can't only compute trust — it has to handle contested trust. Every negative trust effect must be explainable, challengeable, and reversible. No login required.
POST /api/disputes/report
// Report a trust issue — no auth required { "entity_id": "mcp-server-ep-v1", "reason": "demo_challenge", "details": "Interactive demo — testing dispute lifecycle" }
Files a real report (rate-limited)
Constitutional principle: Trust must never be more powerful than appeal. Reports create review objects — they don't directly change trust profiles. That's due process.
5
Explore Trust Policies
EP ships with 8 trust policies across commerce and software. Custom policies are accepted as JSONB. Policies are the decision layer — they define what "trusted enough" means for each context.
GET /api/policies
// List all available trust policies GET /api/policies
Returns all policy definitions
The shift: Agents don't check raw numbers. They evaluate against structured policies. That's how trust becomes context-aware, portable, and machine-actionable.
