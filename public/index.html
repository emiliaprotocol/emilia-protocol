THE PROBLEM

Built for approved-looking workflows
where ordinary auth fails

Benefit payment redirection

An authorized operator changes a payment destination inside a valid session. No control catches the action itself.

Beneficiary and remittance changes

A wire transfer beneficiary is swapped through approved channels. The system sees a legitimate update, not fraud.

Privileged infrastructure actions

A production credential is rotated or a deployment is pushed without action-bound authorization. Access was valid.

AI agent destructive execution

An agent with broad tool access executes a high-risk action. No human assumed responsibility for the specific operation.

PROTOCOL DISCIPLINE

What EP proves before action

01

Actor identity

Cryptographically verified identity of the entity requesting the action.

02

Authority chain

Complete delegation path from root authority to the acting principal.

03

Exact action context

The precise operation, target, parameters, and environmental conditions.

04

Policy version and hash

Immutable reference to the exact policy version that authorized this action.

05

Nonce and expiry

One-time cryptographic nonce and strict temporal bounds on authorization.

06

One-time consumption

Each ceremony token is consumed on use -- no replay, no reuse, no ambiguity.

07

Immutable event traceability

Append-only audit trail linking every authorization to its outcome.

08

Accountable signoff, when required

Named human responsibility for the exact action, cryptographically bound to the ceremony.

HUMAN ACCOUNTABILITY

When policy requires human ownership

EP can require a named responsible human to explicitly assume responsibility for the exact action before execution. The signoff is cryptographically bound to the action context, the policy, and the signer's identity.
